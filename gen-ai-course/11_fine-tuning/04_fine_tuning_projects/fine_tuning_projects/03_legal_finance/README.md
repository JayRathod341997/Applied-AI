# Fine-Tuning for Legal Finance

## Overview

This project demonstrates the fine-tuning of a transformer-based language model on a proprietary collection of legal-finance documents. The goal is to create a domain-aware model capable of generating accurate summaries, extracting structured information, and answering complex queries about financial regulations, contracts, and case law. The fine-tuned model can be integrated into content-generation pipelines, compliance audit tools, and knowledge-base search systems.

## Problem Statement

Financial compliance documents are dense, jargon-heavy, and highly structured. Off-the-shelf language models often misinterpret key clauses, miss regulatory nuances, and fail to maintain consistency across multiple sections. By training on a curated corpus that mirrors real-world legal-finance texts, we reduce these failure modes and provide a tailored NLP engine that respects domain conventions.

## Corpus

The dataset consists of 10,000 annotated PDF and DOCX files spanning 2015-2024. Each document has been converted to plain text, tokenized by a custom spaCy pipeline that preserves named entities, table structures, and numbered clauses. The dataset is split into training, validation, and test sets with a 80/10/10 ratio. Each document is further annotated with:

- Entity tags for regulations (e.g. "SEC", "Dodd-Frank"), company names, and monetary amounts.
- Clause headings and hierarchical numbering.
- Hand-labelled summaries for a subset of 2,000 documents.

## Methodology

The base model is a 24-layer transformer with 16-head self-attention (≈400M parameters). Fine-tuning is performed in two stages:

1. **Domain-specific pre-training** on the legal-finance corpus with masked language modeling to capture industry jargon and sentence-structure patterns.
2. **Task-specific fine-tuning** using supervised learning, employing a combination of cross-entropy loss for next-token prediction and sequence-to-sequence loss for summary generation.

When generating summaries, teacher-forcing is used during training to accelerate convergence. During inference, beam search with a beam width of 4 and a length penalty of 1.2 enhances output diversity while penalizing over-long sentences that lose focus.

## Training

```bash
python -m src.trainer \
  --config configs/legal_finance.yaml \
  --output_dir experiments/2026_04_06 \
  --epochs 12 \
  --batch_size 8 \
  --lr 3e-5
```

The training script pulls hyperparameters from the YAML file, which also specifies data loader parameters, logging frequency, and device allocation. The final checkpoint is automatically uploaded to S3 for HIPAA-compliant backup.

## Evaluation

The model is evaluated on the hold-out test set using the following metrics:

- **ROUGE-L** and **BERTScore** for summary quality.
- **Exact Match** and **F1** for entity extraction.
- **Perplexity** for language modeling.

Results are logged to wandb and summarized in a short PDF report generated by `src/evaluate.py`.

## Inference

```python
from src.inference import generate_summary

doc = open("data/processed/sample_doc.txt").read()
summary = generate_summary(doc, max_new_tokens=150)
print(summary)
```

The inference API accepts a raw text string and returns a concise, style-appropriate summary of the input. For generation of structured data, feed the document to `extract_entities()` in `src.inference`.
