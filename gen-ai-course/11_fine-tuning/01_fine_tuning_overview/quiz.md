# Quiz: Fine-tuning Overview

## Questions

### Question 1
What is the primary purpose of fine-tuning?

A) To create new models from scratch
B) To adapt pre-trained models for specific tasks
C) To reduce model size
D) To increase inference speed

### Question 2
When should you use RAG instead of fine-tuning?

A) When you need consistent output format
B) When you need the latest information
C) When you need offline capability
D) When you want to reduce API costs at inference time

### Question 3
Which is NOT a benefit of fine-tuning?

A) Lower latency
B) Offline capability
C) Always better quality than base model
D) Consistent output format

### Question 4
What is catastrophic forgetting?

A) When the model forgets how to generate text
B) When fine-tuning causes loss of original capabilities
C) When the model becomes too specialized
D) When training data is lost

### Question 5
What is a hybrid approach in LLM applications?

A) Using multiple base models
B) Combining fine-tuning with RAG
C) Using ensemble methods
D) Training on multiple datasets

### Question 6
Which factor favors using fine-tuning over RAG?

A) Need for latest information
B) Limited training data
C) Need for consistent style and format
D) Cost-sensitive application

### Question 7
What is needed to successfully fine-tune a model?

A) Large language model API
B) Sufficient domain-specific training data
C) Vector database
D) Web search capability

### Question 8
Which is typically cheaper: fine-tuning or RAG at inference time?

A) Fine-tuning
B) RAG
C) They are the same
D) Depends on the model size

## Answers

1. B) To adapt pre-trained models for specific tasks
2. B) When you need the latest information
3. C) Always better quality than base model
4. B) When fine-tuning causes loss of original capabilities
5. B) Combining fine-tuning with RAG
6. C) Need for consistent style and format
7. B) Sufficient domain-specific training data
8. A) Fine-tuning
